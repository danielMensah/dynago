## Codebase Patterns
- `Eval(node, item)` evaluates condition/filter AST; `EvalUpdate(nodes, item)` applies update AST returning a new item copy
- `resolvePathValue(parts, item)` traverses nested M attributes for dotted paths; reused by both evaluators
- `resolveNode(node, item)` resolves PathNode/ValueNode/FuncNode(size) to AttributeValue; used in comparisons and updates
- DynamoDB number comparisons: parse string N values to float64 via strconv.ParseFloat
- Update evaluator uses copy-on-write for nested maps to avoid mutating input items
- Package name is `dynago`, all files in project root
- No external dependencies yet; stdlib only
- Tests use standard `testing` package with table-driven tests
- Struct tag name is `dynamo` (not `dynago`); parsed via `parseTag()` in codec.go
- `getCodec(reflect.Type)` returns cached `*typeCodec` via `sync.Map`; use this for marshal/unmarshal
- `fieldOptions` contains all tag metadata including GSI/key info for future phases
- Known tag options: field name, hash, range, gsi:<name>, omitempty, set, unixtime, - (skip)
- `Backend` interface in backend.go defines the 10 DynamoDB methods; all request/response types are library-owned
- `Table.Backend()` and `Table.Name()` are the accessors free functions (Get[T], Query[T]) use
- `KeyValue.Map()` returns the internal map[string]AttributeValue for passing to Backend
- `Key()` panics on misuse (programmer error convention like regexp.MustCompile)
- `Marshal(v any)` in marshal.go converts structs to `map[string]AttributeValue` using cached typeCodec
- `Marshaler` interface (`MarshalDynamo() (AttributeValue, error)`) for custom types
- `marshalValue(reflect.Value, fieldOptions)` is the recursive value converter
- nil pointers are omitted (not encoded as NULL); nil slices/maps encode as NULL
- `isZero()` helper handles omitempty logic for all types including time.Time
- Expression AST lives in `internal/expr/` package; Node interface has private marker method
- `ParseCondition(expr, args...)` parses placeholder expressions into AST with correct precedence (NOT > AND > OR)
- `BuildSet/BuildAdd/BuildRemove/BuildDelete` construct UpdateNode for update expressions
- `goToAttributeValue(v any)` in builder.go converts Go values to dynago.AttributeValue
- Reserved words in `internal/reserved.go` as `map[string]bool` with uppercase keys; use `IsReserved(word)` for lookup
- `ToDynamo(node)` returns (exprString, names, values); `ToDynamoUpdates([]UpdateNode)` groups by action; `ToDynamoProjection(node)` for projections
- IMPORT CYCLE: `internal/expr` imports `dynago` (for AttributeValue in ValueNode). Do NOT import `internal/expr` from the `dynago` package. Build projection/condition expressions inline instead.
- `buildProjection(attrs)` in get.go builds projection expressions without importing internal/expr
- All attribute names get `#name` aliases in expression translation, not just reserved words
- Value placeholders use `:v0`, `:v1`, etc. with incrementing counter
- Logical AND/OR wrap operands in parens: `(left) AND (right)`; NOT wraps operand: `NOT (operand)`
- `Unmarshal(item, &out)` in unmarshal.go decodes `map[string]AttributeValue` into structs using cached typeCodec
- `Unmarshaler` interface (`UnmarshalDynamo(AttributeValue) error`) for custom types, checked via pointer receiver first
- `unmarshalValue(av, dst, opts)` is the recursive value decoder; handles pointers, sets, maps, nested structs
- `reflect.TypeFor[T]()` preferred over `reflect.TypeOf((*T)(nil)).Elem()` for type comparisons
- Iterators use `iter.Seq2[T, error]` (Go 1.23 range over func) — no .Next()/.Err() pattern
- `conditionExpr` struct in condition.go holds expression string + names + values maps
- `buildCondition(expr, args)` and `mergeConditions` combine expressions with AND
- `anyToAttributeValue(v any)` converts Go values to AttributeValue (used by condition builder)
- `noopBackend` pattern: implement all Backend interface methods, store pre-built responses, return them without doing work
- `stubBackend` pattern: embed noopBackend, override specific methods, capture calls slice for assertions
