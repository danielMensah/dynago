## Codebase Patterns
- Package name is `dynago`, all files in project root
- No external dependencies yet; stdlib only
- Tests use standard `testing` package with table-driven tests
- Struct tag name is `dynamo` (not `dynago`); parsed via `parseTag()` in codec.go
- `getCodec(reflect.Type)` returns cached `*typeCodec` via `sync.Map`; use this for marshal/unmarshal
- `fieldOptions` contains all tag metadata including GSI/key info for future phases
- Known tag options: field name, hash, range, gsi:<name>, omitempty, set, unixtime, - (skip)
- `Backend` interface in backend.go defines the 10 DynamoDB methods; all request/response types are library-owned
- `Table.Backend()` and `Table.Name()` are the accessors free functions (Get[T], Query[T]) use
- `KeyValue.Map()` returns the internal map[string]AttributeValue for passing to Backend
- `Key()` panics on misuse (programmer error convention like regexp.MustCompile)
- Expression AST lives in `internal/expr/` package; Node interface has private marker method
- `ParseCondition(expr, args...)` parses placeholder expressions into AST with correct precedence (NOT > AND > OR)
- `BuildSet/BuildAdd/BuildRemove/BuildDelete` construct UpdateNode for update expressions
- `goToAttributeValue(v any)` in builder.go converts Go values to dynago.AttributeValue
- Reserved words in `internal/reserved.go` as `map[string]bool` with uppercase keys; use `IsReserved(word)` for lookup
